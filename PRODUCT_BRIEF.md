# Product Brief

## Target user

A product, growth, or merchant-success operator responsible for converting shopper intent into measurable merchant value.

## Problem

Wishlist, save-for-later, and back-in-stock events often exist as disconnected signals. Operators must determine which signal is actionable, confirm the supporting product state, prepare an activation, preserve review boundaries, and measure incremental impact.

## Product objective

Turn a raw intent signal into an inspectable decision:

`source records -> deterministic opportunity -> agent analysis -> human decision -> controlled activation -> holdout attribution`

## User decisions

- Is the signal complete and current enough to act on?
- Which shoppers are eligible?
- What value assumption is being used?
- What should the merchant do?
- What information is missing?
- Should the recommendation be approved, edited, or rejected?
- Did the action produce incremental value?

## Success measures

- Every opportunity links to source evidence.
- Unsupported numerical claims are rejected.
- Human review remains mandatory.
- A cloned repository can generate a fresh agent result without an API key.
- Attribution distinguishes treatment performance from baseline behavior.

## Non-goals

- Sending live merchant campaigns.
- Predicting conversion with a trained model.
- Reproducing any proprietary commerce platform.
- Claiming synthetic outcomes as real customer performance.
- Providing production authentication, storage, or consent management.
